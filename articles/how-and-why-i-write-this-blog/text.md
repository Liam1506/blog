The first question that comes to mind is why I am writing my blog here with my own system instead of using Medium or dev.to. The main reason is that I recently bought the domain [wttg.li](https://wttg.li), but at the moment, I don't really have a purpose for it. So I thought, why not set up a blog here?

The next question is why I don't use an already created tool like [Wordpress](https://wordpress.com/) or [Ghost](https://ghost.org/). The main reason is that I just like building things. I know it’s objectively a bad decision, but I enjoy it. Moreover, I don't really like the "setup hustle" that comes with big CMS tools, although Ghost isn't that bad from all I’ve heard. I also recently saw that you can use the Astro blog template and probably get a better result, but where's the fun in that?

### How I Built This Blog

The tech stack I settled on was a Python static page generator that generates plain HTML and CSS. No framework, no server. I decided to use Python because I haven't used it in a long time and just wanted to refresh my memory a bit. I enjoy Python quite a bit, though I forgot how "interesting" a non-type-safe language can be.

For the aesthetic, I tried to go for a Hacker News-like style. I know it’s not exactly pretty, but I enjoy surfing there and like the community itself. This style also aligns very well with my design decision to only use plain HTML and CSS.

For hosting, I decided to use GitHub Pages. It’s incredibly easy to use since I can just push my repository to update the blog. Later on, I will move the compile script to GitHub Actions, but I haven't found the time yet, so for now, I compile the blog on my laptop. That’s basically all the tech I used. The whole project is <300 lines of Python code.

### What I Will Do on This Blog

TBH, I'm not really sure right now. Like I mentioned, I don't have any specific plans, nor do I plan to add any features right now. I guess I will just post once in a while about things I find interesting.

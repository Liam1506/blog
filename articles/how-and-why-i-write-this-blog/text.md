# How and why I write this blog
2026-02-24 10:02

In this case writing has here more like a double meaning. On the one hand I will write about, how I wrote this blog itself and set it up (it's nothing crazy). In the second path I want to write more about my motivation creating a blog in the first place. 

### Why and how I wrote the blog itself.
The first question that comes to mind is, why I write my blog here with my own blog system instead of medium or dev.to. The main reason is, that I recently bought the domain [wttg.li](https://wttg.li) but at the moment I don't have really a purpose for it. So I thought, why not set up a blog here. The next question that comes to mind, why I don't use an already created Blog tool like [Wordpress](https://wordpress.com/) or [ghost](https://ghost.org/). The main reason is that I like building things. I know, it's objectively a bad decision, but I kinda enjoy it. Moreover, I don't really like the setup hustle, that comes with big CRM tools, although [ghost](https://ghost.org/) isn't that bad form all I heard.

### How I build this blog
One thing that is very important to me, is that the blog is maintainable. What I mean with this isn't the code itself, but more like the content posting. I juts want an easy way to create articles and remove them. So my eyes fell on git. I'm personally a very big git fan. I use it all the time at work and I love it. Additonaly I have GitHub pro, because I'm in University, so I can even create a GihtubPage from a private repository. So, I create a few python scripts, that create html from md. I guess, when you add all lines up, you are below 200 lines of code. It took me like 4 hours.
